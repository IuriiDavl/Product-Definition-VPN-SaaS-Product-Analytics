# Product Definition — Аналитика VPN SaaS-продукта

## Обзор проекта

Данный проект моделирует продуктовую аналитику подписочного VPN SaaS-сервиса на основе event-based e-commerce датасета.

Несмотря на то, что исходный датасет относится к электронной коммерции, пользовательские события переосмыслены как события взаимодействия с SaaS-продуктом. Цель проекта — провести анализ поведения пользователей, активации, конверсии, retention и вовлечённости с использованием funnel и cohort analysis.

---

# Бизнес-проблема

VPN-платформа теряет пользователей на разных этапах пользовательского пути:

- пользователи заходят в продукт, но не начинают пользоваться
- пользователи проявляют интерес, но не оформляют подписку
- часть пользователей покупает подписку только один раз
- retention и повторное использование продукта остаются низкими

Цель проекта — определить:

- на каких этапах пользователи отваливаются
- какие действия влияют на конверсию
- как retention меняется между когортами
- чем удержанные пользователи отличаются от ушедших

---

# Описание продукта

Продукт представляет собой подписочный VPN SaaS-сервис, который позволяет пользователям:

- использовать защищённое интернет-соединение
- подключаться к VPN-серверам
- использовать шифрование трафика
- оформлять премиум-подписку
- продлевать подписку и регулярно пользоваться сервисом

Бизнес-модель построена по SaaS-принципу с акцентом на удержание пользователей и регулярное использование продукта.

---

# Определение продуктовой воронки

Продуктовая воронка моделируется следующим образом:

1. Привлечение пользователя
2. Ознакомление с продуктом
3. Активация
4. Конверсия
5. Удержание
6. Лояльность / повторное использование

---

# Интерпретация событий (Event Mapping)

События исходного датасета переосмыслены как события SaaS-продукта.

| Событие датасета | Интерпретация в SaaS | Этап воронки |
|---|---|---|
| view | Открытие приложения / посещение продукта | Привлечение / Ознакомление |
| addtocart | Начало trial / намерение оформить подписку | Активация |
| transaction | Покупка платной подписки | Конверсия |
| repeated transaction | Продление подписки | Удержание / Лояльность |

---

# Определение активации

Активация — это первое значимое действие пользователя внутри продукта.

В рамках проекта активацией считается:

- первое событие `addtocart`
- либо повторное взаимодействие после первого посещения

Активация показывает, что пользователь заинтересовался продуктом и начал двигаться к получению ценности от сервиса.

---

# Определение retention

Retention измеряется через повторную активность пользователей после первой активации или конверсии.

Примеры удержанного поведения:

- повторные сессии
- повторные взаимодействия
- повторные покупки подписки
- регулярная активность во времени

Retention анализируется через:

- недельные когорты
- месячные когорты
- retention curves
- анализ повторного поведения

# Ключевые продуктовые метрики

Проект фокусируется на следующих продуктовых метриках:

## Метрики воронки
- conversion rate
- activation rate
- drop-off rate
- конверсия между этапами

## Метрики вовлечённости
- active users over time
- частота повторного использования
- активность пользователей

## Метрики удержания
- D1 / D7 / D30 retention
- cohort retention
- returning users

## Метрики монетизации
- конверсия в покупку
- повторные покупки
- сегменты лояльных пользователей

# Сегментация пользователей

Пользователи будут разделены на поведенческие сегменты:

- новые пользователи
- активированные пользователи
- пользователи с конверсией
- удержанные пользователи
- ушедшие пользователи
- лояльные пользователи

---

# Цели аналитики

Проект должен ответить на следующие бизнес-вопросы:

1. На каких этапах пользователи отваливаются?
2. Какие действия влияют на конверсию?
3. Какие когорты удерживаются лучше всего?
4. Как вовлечённость влияет на retention?
5. Чем лояльные пользователи отличаются от ушедших?
